# 殡仪馆地下二层 - 游戏说明

## 🏛️ 游戏简介

这是一款发生在殡仪馆地下二层的H5经营解谜游戏。你是一名遗物处理师，白天接单，夜里处理那些"不太对劲"的遗物。

**技术栈**：Vue 3 + Canvas API + Pinia + Vite + TypeScript + Tailwind CSS

**游戏特色**：
- 昼夜交替的经营玩法
- 8种不同类型的灵异遗物，每种都有独特的背景故事
- Canvas绘制的动态遗物效果
- 随机触发的异常事件和剧情选择
- 理智值系统，感受逐步陷入疯狂的体验
- Web Audio API生成的氛围音效
- 本地存档功能

---

## 🎮 基本玩法

### 1. 昼夜系统

**☀️ 白天（8:00 - 18:00）**
- 查看待接订单列表
- 接受或拒绝订单
- 无法处理遗物

**🌙 夜晚（18:00 - 03:00）**
- 在工作台上处理遗物
- 进行清洁、修复、净化、封存四步流程
- 可能触发灵异事件

点击顶部的"进入夜晚/结束夜晚"按钮切换时段。

### 2. 接单流程

1. 白天查看左侧订单面板中的"待接订单"
2. 每个订单显示：
   - 委托人信息、逝者信息
   - 遗物类型和异常类型
   - 难度等级（★☆☆ ~ ★★★）
   - 报酬（金钱 + 声望）
3. 点击"接受"接下订单，或"拒绝"跳过

> 💡 **提示**：至少接受一个订单才能进入夜晚。

### 3. 遗物处理

1. 夜晚时，在左侧"待处理"列表中选择一个遗物
2. 查看遗物信息和背景故事
3. 按照步骤依次处理：
   - **🧼 清洁**：用特制溶液清洁表面
   - **🔧 修复**：修复破损之处
   - **✨ 净化**：用香薰和符咒净化阴气
   - **📦 封存**：放入特制收纳盒
4. 点击"开始"按钮或直接点击Canvas区域开始处理
5. 每步完成后，可能触发灵异事件
6. 所有步骤完成后，点击"完成订单"获取奖励

### 4. Canvas交互

工作台上的Canvas区域会：
- 实时绘制当前选中的遗物
- 显示处理进度条
- 鼠标悬停显示选择圈
- 低理智值时出现画面抖动和红光闪烁
- 遗物有轻微的呼吸和旋转动画

## 📊 数值系统

### 💰 金钱
- 初始值：2000
- 完成订单获得报酬
- 部分选择事件会消耗金钱
- 用于维持运营

### ⭐ 声望
- 初始值：50（范围：0 ~ 100）
- 完成订单获得声望奖励
- 失败订单扣除声望
- 部分选择事件影响声望
- 影响后续订单的质量和数量

### 🧠 理智值
- 初始值：100（范围：0 ~ 100）
- 处理遗物会消耗理智
- 触发灵异事件会扣除理智
- 每天白天恢复10点
- **归零则游戏结束**

> ⚠️ **警告**：理智值低于30时，画面会出现异常效果，同时会听到心跳声。

## 👻 事件系统

### 异常事件（Anomaly）
- 处理遗物时随机触发
- 表现为各种灵异现象
- 扣除理智值
- 伴有特殊音效

### 选择事件（Choice）
- 特定天数或随机触发
- 需要做出道德选择
- 每个选择影响不同数值
- 有些选择会影响剧情走向

### 剧情事件（Narrative）
- 固定天数触发
- 讲述地下二层的秘密
- 第3、7、14天有特殊剧情

---

## 🎵 音频系统

游戏使用 Web Audio API 实时生成所有音效，无需外部音频文件。

### 氛围音效
- 低频嗡鸣 + 白噪音模拟地下室环境
- 轻微的音量波动增加不安感

### 交互音效
- 点击音、成功提示音
- 处理时的工作音效（不同步骤不同音色）
- 灵异事件时的扭曲音效
- 低语声、心跳声等恐怖音效

### 操作
- 点击右上角🔊按钮切换静音
- 菜单中可手动初始化音频（浏览器策略要求用户交互后才能播放音频）

---

## 💾 存档系统

- 自动检测本地存档
- 开始界面可选择"继续游戏"
- 菜单中可手动保存
- 存档内容：天数、金钱、声望、理智值、时间阶段
- 存档存储在浏览器 localStorage 中

---

## 🎯 游戏目标

1. 尽可能多地处理订单，积累金钱和声望
2. 保持理智值不耗尽
3. 探索所有遗物的背景故事
4. 存活尽可能多的天数
5. 发现地下二层的秘密

## 🛠️ 开发说明

### 项目结构
```
src/
├── components/          # Vue组件
│   ├── OrderPanel.vue   # 订单面板
│   ├── Workbench.vue    # 工作台（Canvas）
│   ├── EventModal.vue   # 事件弹窗
│   └── StatusBar.vue    # 状态栏
├── game/                # 游戏核心
│   ├── types.ts         # 类型定义
│   ├── audio.ts         # 音频系统
│   └── data/            # 游戏数据
│       ├── relics.ts    # 遗物数据
│       └── events.ts    # 事件脚本
├── stores/              # Pinia状态管理
│   ├── gameStore.ts     # 游戏核心状态
│   ├── orderStore.ts    # 订单管理
│   └── eventStore.ts    # 事件管理
└── pages/
    └── GamePage.vue     # 主游戏页面
```

```bash
npm install    # 安装依赖

npm run build  # 构建生产版本

```

## ⚠️ 注意事项

1. **浏览器兼容性**：需要支持 Canvas API 和 Web Audio API 的现代浏览器
2. **音频播放**：由于浏览器策略，首次点击后才会播放音频
3. **存档**：清除浏览器数据会丢失存档
4. **内容提示**：游戏包含恐怖元素和心理暗示，敏感人群请谨慎游玩

---

## 🏁 游戏结束条件

1. **理智值归零**：被地下的存在吞噬了意识
2. **破产 + 低声望**：没钱继续经营，名声也臭了
